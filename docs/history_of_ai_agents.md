# AIコーディングエージェントの歴史と進化

AIによるコーディング支援は、この数年で単なる補完ツールから、コード理解・編集・実行まで担うエージェントへと急速に進化しました。本稿では、現在の選定候補を理解しやすくするため、その流れを形づくった近年の2つの潮流に絞って整理します。

---

## 1. 対話型コーディング支援とAIネイティブIDEの登場（2021年〜2024年）

大規模言語モデルの進化により、コードの「補完」から自然言語による「対話型支援」へと重心が移りました。その流れの中で、既存IDEに統合されるコパイロット型ツールや、コードベース全体の理解を前提にしたAIネイティブIDE、自律的に動くエージェントが登場しました。

*   **GitHub Copilot in VS Code (2021年プレビュー、2022年一般公開)**
    OpenAIのCodexを基盤とした代表的なコパイロット型ツールです。初期はVS Code上でのコードサジェストや補完が中心でしたが、コメントや関数シグネチャからコードを生成する体験を広く普及させ、その後のCopilot Chatによって対話型の開発支援を一般化しました。
*   **Cursor (2023年〜)**
    VS Codeをフォークして開発されたAIネイティブIDEです。初期は強力なサジェストやインライン補完の体験でも注目されましたが、その後は単一ファイルの編集を超え、複数ファイルにまたがる変更を一括で行う「Composer」や、プロジェクト特有のルールをAIに記憶させる「Memories」機能など、コンテキストを深く理解したコーディング体験へと拡張しました。
*   **Devin (2024年3月発表)**
    Cognition Labsが発表した「世界初の完全自律型AIソフトウェアエンジニア」。要件を伝えると、計画の立案、コードの記述、ブラウザを使ったドキュメントの検索、テストの実行、バグの修正までを自律的に完結させます。SWE-bench等のベンチマークで記録的なスコアを叩き出し、AIエージェントの新たな到達点を示しました。

## 2. CLIベースのエージェントと開発ワークフローの統合（2025年〜）

IDEのGUIに依存せず、開発者が日常的に使用するターミナル（CLI）上で直接機能し、より柔軟にOSのコマンドや各種ツールと連動するエージェントが台頭しています。

*   **Claude Code**
    AnthropicによるターミナルネイティブなAIエージェント。ターミナル上で対話しながら、コードベースの広範な調査、ファイルの編集、bashコマンド（gitやテストコマンドなど）の実行をシームレスに行うことができます。自律的な計画立案モード（Plan Mode）などを備え、複雑なリファクタリングやバグ修正に強みを持ちます。
*   **Codex CLI**
    OpenAIによるターミナル向けのオープンソースなコーディングエージェントです。ローカルのワークフローに組み込みやすく、リポジトリの調査、ファイル編集、コマンド実行を一体で扱える点が特徴です。
*   **GitHub Copilot CLI**
    GitHub Copilotをターミナルから利用できるCLIです。自然言語でコードの理解、修正、デバッグを進められ、GitHub連携や対話型の作業フローをCLIに持ち込める点が特徴です。現在はパブリックプレビューとして提供されています。
*   **Gemini CLI**
    Googleのモデルを活用し、CLI環境に統合されたコーディングエージェント。数百万トークンという巨大なコンテキストウィンドウを活かし、大規模なリポジトリ全体を一度に読み込んで理解し、複雑なタスクをCLI上で自律的かつ連続的に処理できる点が特徴です。
*   **Cursor CLI**
    Cursorのエージェント機能をターミナルから使えるCLIです。対話型セッションだけでなく、スクリプトやCI向けの非対話実行にも対応し、既存のIDE運用とCLI自動化をまたいで使える点が特徴です。現在はベータとして案内されています。

---

## まとめ

近年のAIコーディング支援の進化は、補完中心のコパイロット型から、自然言語で対話しながらコードベースを理解し、複数ファイルの編集やコマンド実行まで担うエージェント型へと重心が移ってきた点にあります。
特に直近では、AIネイティブIDEに加えてCLIベースのエージェントが登場し、既存の開発ワークフローにより深く組み込めるようになりました。CLI型は `git` `test` `lint` `terraform` `aws` などの既存CLI資産をそのまま呼び出せるため、自動化、CI連携、監査ログ取得、実行権限の統制に載せやすく、エンタープライズ運用との相性がよい点が強みです。また、GUIに依存せず軽量に起動・操作できるため、IDE全体を立ち上げるよりも素早く試行錯誤しやすく、調査・修正・検証の往復を高速に回しやすい傾向があります。一方で、IDE型は補完や可視化で優位な場面もありますが、今回の選定では「どこまで自律的に実行できるか」と「日常の開発環境にどれだけ自然に統合できるか」に加え、「どれだけ高速かつ再現可能に運用へ載せられるか」が重要な比較軸になります。
